# v0.2 范围 — Roguelike 循环

## 概述

v0.2 将原型扩展为完整的 Roguelike 循环: 程序化房间生成、通关后 3 选 1 能力、Boss 战、元进度和运行统计。

---

## 范围

### 运行管理器 (RunManager)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 运行生命周期 | 开始/推进/结束运行 | P0 |
| 房间推进 | 清理房间 -> 选择路径 -> 加载下一个 | P0 |
| 运行结构 | 神庙 -> 5 个房间 -> Boss | P0 |
| 游戏结束 | 死亡处理, 运行统计汇总 | P0 |

### 能力系统 (AbilitySystem)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 能力选择 | 房间清理后 3 选 1 | P0 |
| 能力堆叠 | 相同能力可堆叠增强 | P0 |
| 能力池 | 初始 6+ 个能力 | P0 |
| 能力效果 | 伤害加成、速度加成、生命恢复、额外冲刺等 | P0 |
| 能力 UI | 选择界面, 当前能力显示 | P0 |

### 程序化生成 (ProceduralGeneration)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 房间布局生成 | 随机房间形状, 墙壁/地板 TileMap | P0 |
| 房间模板库 | 5+ 个房间模板 (战斗/奖励/商店/休息/精英) | P0 |
| 敌人放置 | 根据房间类型和难度生成敌人 | P0 |
| 路径选择 | 2-3 条路径分支 | P1 |

### 新敌人

| 敌人 | 描述 | 优先级 |
|------|------|--------|
| Bone Archer | 远程敌人, 射箭, 保持距离 | P0 |
| Slime | 分裂敌人, 死亡时分裂为 2 个小史莱姆 | P0 |

### Boss

| Boss | 描述 | 优先级 |
|------|------|--------|
| Shadow Lord | 2 阶段, 阶段 1: 暗影球 + 冲刺, 阶段 2: 召唤 + AOE | P0 |
| Boss 房间 | 专门 Boss 战场地 | P0 |
| Boss UI | Boss 生命值条 | P0 |

### 元进度 (MetaProgression)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 灵魂货币 | 运行中获得灵魂, 永久保留 | P0 |
| 解锁加成 | 灵魂消耗解锁永久加成 | P0 |
| 加成种类 | 初始 HP+、移速+、攻击+、冲刺+ | P0 |

### 运行统计 (RunStats)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 运行统计 | 击杀数、伤害输出、承受伤害、房间清完数 | P0 |
| 运行摘要 | 死亡后显示运行摘要 | P0 |
| 历史记录 | 最佳运行记录 | P1 |

## 技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 房间模板 | PackedScene 资源 | 编辑器可视化编辑 |
| 程序化布局 | 基于 TileMap 的算法 | 与现有渲染兼容 |
| 能力数据 | Resource 文件 | 数据驱动, 易于扩展 |
| 元数据 | ConfigFile 或 JSON | 简单持久化 |
| 随机种子 | 基于 Godot 的 RNG | 确定性随机 |

---

## 不包括范围 (v0.2)

- 本地化
- 音效/音乐
- 成就系统
- 每日挑战
- 在线排行榜
- 多语言支持
