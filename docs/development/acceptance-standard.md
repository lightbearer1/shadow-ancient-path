# 验收标准

> 此文件为只读文件，任何情况下不得降低标准。

## 性能标准

| 指标 | 标准 | 测试方法 |
|------|------|----------|
| 帧率 | 60 FPS (目标), 30 FPS (最低) | Profiler 运行5分钟 |
| 启动时间 | < 3秒 (从启动到主菜单) | 计时器测量 |
| 内存使用 | < 256 MB (峰值) | 任务管理器 / Godot Profiler |
| CPU使用率 | < 30% (平均) | Profiler |
| 场景切换 | < 0.5秒 | 计时器测量 |

## 功能标准

### 玩家 (v0.1)

| 功能 | 标准 |
|------|------|
| 移动 | 水平移动响应时间 < 50ms |
| 跳跃 | 跳跃高度 >= 48px, 可二段跳跃 (如有) |
| 冲刺 | 冲刺距离 >= 96px, 冷却时间 <= 1s |
| 3连击 | 完整连击动画 <= 1.5s, 输入缓冲窗口 200ms |
| 死亡 | 死亡动画后 2s 触发游戏结束 |

### 敌人 (v0.1)

| 功能 | 标准 |
|------|------|
| 巡逻 | 在两个点之间移动, 速度 <= 玩家移动速度的 60% |
| 追击 | 检测范围 >= 120px, 追击速度 >= 巡逻速度的 1.5x |
| 攻击 | 攻击前摇 <= 0.5s, 攻击范围 <= 32px |
| 死亡 | 死亡动画 <= 1s, 击杀计数更新 |

### 战斗 (v0.1)

| 功能 | 标准 |
|------|------|
| 命中检测 | 攻击帧命中框与实际碰撞体偏差 < 4px |
| 伤害计算 | 基础攻击伤害 >= 1, 连击倍率准确 |
| 连击计时 | 连击超时 = 1.5s, 超时后重置 |
| 击杀 | 击杀后 0.5s 内更新 UI |

### UI (v0.1)

| 功能 | 标准 |
|------|------|
| 主菜单 | 按 "开始游戏" 后 < 0.5s 加载关卡 |
| HUD | 生命值和连击显示更新间隔 < 100ms |
| 游戏结束 | 死亡后 < 2s 显示, 可选择重试/返回菜单 |

## 测试标准

| 指标 | 标准 |
|------|------|
| 总覆盖率 | >= 80% (行覆盖率) |
| 玩家系统覆盖率 | >= 85% |
| 敌人系统覆盖率 | >= 80% |
| 战斗系统覆盖率 | >= 90% |
| UI系统覆盖率 | >= 70% |
| 状态转换 | 所有状态转换必须被测试 |
| 边界条件 | 所有伤害计算必须有边界测试 (0, 最大值, 浮点数精度) |

## 代码标准

| 规则 | 标准 |
|------|------|
| 硬编码值 | 无硬编码值，所有可调参数使用 @export 或常量 |
| 信号 | 所有信号必须有类型定义 (signal signal_name(param: Type)) |
| 返回值 | 所有函数必须有返回值类型标注 |
| 占位符 | 无 pass 占位符，每个函数必须有实现 |
| 文件长度 | 每个文件 < 400 行 |
| 函数长度 | 每个函数 < 30 行 |
| 嵌套层级 | 不超过 4 层 |

## 不可降低的标准

以下标准在任何情况下不得降低:

1. 60 FPS 目标帧率
2. 80% 最低测试覆盖率
3. 所有伤害计算的边界测试
4. 所有状态转换的测试覆盖
5. 无硬编码值 (使用 @export 或常量)
6. 无 pass 占位符
7. 所有信号必须有类型定义
